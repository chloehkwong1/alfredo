Worktrees from all selected repos sorted together by status column. Small colored tag on each item shows which repo. Good for: "show me everything that needs attention across all repos."

B — Repo Sections

Multi-select pills at top. Worktrees grouped under repo headers (each collapsible). Each repo gets its own color. Status columns still exist within each repo group. Good for: "I think of my repos as separate projects."

The key trade-off

A — Mixed by status

  • See all "needs attention" across repos at once
  • Status-first thinking: "what needs me?"
  • Compact — no repeated section headers

B — Grouped by repo

  • Clear mental model: repos are separate
  • Repo-first thinking: "what's happening in X?"
  • Each repo can be collapsed independently
