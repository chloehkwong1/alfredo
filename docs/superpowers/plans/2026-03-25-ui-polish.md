## Requirements

> The current UI has four core issues: flat visual hierarchy (sidebar/tabs/status bar share the same background), narrow typography scale (10–14px only), inconsistent spacing rhythm (ad-hoc mix of values), and generic personality. This plan addresses all four by introducing design tokens, background zones for visual hierarchy, a responsive density system, and a new onboarding configuration step. All changes must work across all 8 existing themes.

## Acceptance Criteria

- [ ] Sidebar background is visibly darker than the content area across all 8 themes
- [ ] Sidebar header and tab bar are both 40px tall with bottom borders aligned as one continuous line
- [ ] Status bar is 32px tall with top border aligned with sidebar footer border
- [ ] All spacing in modified components uses the defined spacing scale (4, 8, 12, 16, 24px) — no arbitrary values
- [ ] Agent items use inset card style with 8px margin from sidebar edge and 8px border-radius
- [ ] Responsive density works: compact (< 900px), default (900–1439px), comfortable (≥ 1440px)
- [ ] Sidebar auto-collapses at < 700px window width
- [ ] Onboarding Step 2 shows GitHub token, setup scripts, and worktree location fields
- [ ] Onboarding Step 2 writes config values to the existing `AppConfig` via `saveConfig`
- [ ] "Workspace settings" text link removed from sidebar footer
- [ ] Dialog padding updated to 28px with larger title size (15px semibold)

## Decisions

- One unified plan (not split into sub-plans) since all changes share the token foundation
- Onboarding Step 2 includes full save integration to `AppConfig` store
- Tests in a dedicated final section
- `--bg-sidebar` and `--bg-bar` tokens expressed as relative adjustments per theme (not hardcoded hex)
- Density observer uses `ResizeObserver` + `data-density` attribute (not CSS media queries) for JS-level awareness
- `worktreeBasePath` added to `AppConfig` type for the onboarding worktree location field

### Files to read:
- `src/components/layout/AppShell.tsx` — Where to call the hook
- `src/stores/workspaceStore.ts` — Sidebar collapse state (for auto-collapse at < 700px)

### Changes:

**src/hooks/useDensity.ts** — New hook:

```typescript
import { useEffect } from "react";
import { useWorkspaceStore } from "../stores/workspaceStore";

type Density = "compact" | "default" | "comfortable";

function getDensity(width: number): Density {
  if (width < 900) return "compact";
  if (width >= 1440) return "comfortable";
  return "default";
}

export function useDensity() {
  const setSidebarCollapsed = useWorkspaceStore((s) => s.setSidebarCollapsed);

  useEffect(() => {
    const root = document.documentElement;

    function update() {
      const width = window.innerWidth;
      const density = getDensity(width);
      root.setAttribute("data-density", density);

      // Auto-collapse sidebar at narrow widths
      if (width < 700) {
        setSidebarCollapsed(true);
      }
    }

    update();

    const observer = new ResizeObserver(() => update());
    observer.observe(root);

    return () => observer.disconnect();
  }, [setSidebarCollapsed]);
}
```

**src/styles/globals.css** — Add density-responsive CSS overrides after the `@theme {}` block:

```css
/* ── Density-responsive spacing ──────────────── */
:root {
  --space-component: var(--space-md);   /* 12px default */
  --space-section: var(--space-lg);     /* 16px default */
}

:root[data-density="compact"] {
  --space-component: var(--space-sm);   /* 8px */
  --space-section: var(--space-md);     /* 12px */
}

:root[data-density="comfortable"] {
  --space-component: var(--space-lg);   /* 16px */
  --space-section: var(--space-xl);     /* 24px */
}
```

**src/components/layout/AppShell.tsx** — Add hook call near top of component:

